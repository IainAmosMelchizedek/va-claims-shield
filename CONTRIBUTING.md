# Contributing to VA Claims Shield

First — thank you. This project exists because veterans are tired of fighting alone.

## Before You Start

1. Check the Issues tab to see what is open
2. Comment on the issue you want to work on before starting — this prevents duplicate work
3. Fork the repository and create a branch named after the issue (e.g. `feature/submission-tracker`)

## Pull Request Requirements

- Describe what you changed and why
- Test your changes before submitting
- One feature or fix per pull request — keep it focused

## Opening Issues

If you are a veteran describing a problem you have experienced with the VA:
- Be as specific as possible about what happened
- Describe what a tool could have done to help
- Your experience directly shapes what we build next

## Code Standards

- Node.js / React only (no framework switching without discussion)
- Every function must be tested before it is considered done
- No hardcoded API keys or credentials anywhere in the codebase
- Use environment variables for all sensitive values

## Questions

Open a Discussion on GitHub. We will respond.
